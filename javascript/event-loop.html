JavaScript Event Loop Explained

Understanding how JavaScript handles asynchronous operations

What is the Event Loop?

The event loop is JavaScript's mechanism for handling asynchronous operations. Despite being single-threaded, JavaScript can perform non-blocking operations through the event loop.

How the Event Loop Works

JavaScript has a call stack, a task queue (macrotask queue), a microtask queue, and an event loop that coordinates everything.

Call Stack

Where function calls are executed in a Last-In-First-Out (LIFO) manner.

Web APIs

Browser-provided APIs for asynchronous operations (setTimeout, fetch, DOM events).

Task Queues

Microtasks
Macrotasks
Animation Frames

Microtasks have higher priority than macrotasks. The event loop processes all microtasks before moving to the next macrotask.

Event Loop

Continuously checks if the call stack is empty. If it is, it takes the first task from the queue and pushes it to the call stack.

Key Point: JavaScript is single-threaded, but the browser provides Web APIs that run on separate threads. The event loop coordinates between the call stack and these APIs.

Execution Order

The event loop follows this general order:

  1. Execute all synchronous code
  2. Execute all microtasks (Promises, queueMicrotask)
  3. Execute one macrotask (setTimeout, setInterval, I/O)
  4. Check if rendering is needed (animation frames)
  5. Repeat from step 2

This example demonstrates what happens when you block the event loop with synchronous code.
