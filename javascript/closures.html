JavaScript Closures Explained

Understanding how closures work in JavaScript with interactive examples

What is a Closure?

A closure is a function that has access to variables from its outer (enclosing) scope even after the outer function has finished executing.

How Closures Work

When a function is declared, it captures a reference to its surrounding lexical environment (the scope chain). This environment includes any variables that were in scope at the time the function was created.

// Example of a simple closure function outerFunction(outerVar) { // innerFunction is a closure return function innerFunction(innerVar) { return outerVar + innerVar; }; } const closureExample = outerFunction(5); console.log(closureExample(3)); // Output: 8

In this example, innerFunction "closes over" the outerVar variable, preserving access to it even after outerFunction has finished executing.

Key Point: Closures don't store the actual values of variables, but rather references to them. If the variable changes, the closure will see the updated value.

Interactive Examples

Example 1: Counter with Closure

This example demonstrates how closures can be used to create private variables.

function createCounter() { let count = 0; return function() { count++; return count; }; } const counter = createCounter();

Example 2: Custom Greeting

This example shows how closures can remember values from their creation context.

function createGreeting(greeting) { return function(name) { return `${greeting}, ${name}!`; }; } const sayHello = createGreeting("Hello"); const sayHi = createGreeting("Hi");

Example 3: Private Variables

This example demonstrates how closures can create truly private variables in JavaScript.

function createBankAccount(initialBalance) { let balance = initialBalance; return { deposit: function(amount) { balance += amount; return balance; }, withdraw: function(amount) { if (amount <= balance) { balance -= amount; return balance; } else { return "Insufficient funds"; } }, getBalance: function() { return balance; } }; } const account = createBankAccount(100);

Common Closure Pitfall

This example shows a common mistake when using closures in loops.

// Problem: All functions log the same value for (var i = 0; i < 3; i++) { setTimeout(function() { console.log(i); // Outputs: 3, 3, 3 }, 100); } // Solution: Use let or create a new scope for (let i = 0; i < 3; i++) { setTimeout(function() { console.log(i); // Outputs: 0, 1, 2 }, 100); }
