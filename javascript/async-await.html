Microtask Queue & Event Loop

Execution Order Priority

Promises execute in the Microtask Queue, which has higher priority than the Callback Queue (setTimeout, DOM events).

Event Loop Example 
console.log('1: Sync');

setTimeout(() => console.log('2: Timeout'), 0);

Promise.resolve()
  .then(() => console.log('3: Promise'));

console.log('4: Sync');

// Output order:
// 1: Sync
// 4: Sync
// 3: Promise  (Microtask)
// 2: Timeout  (Macrotask)

Promise Composition

Static Methods

// Wait for all promises
await Promise.all([p1, p2, p3]);

// First to settle
await Promise.race([p1, p2, p3]);

// First to fulfill
await Promise.any([p1, p2, p3]);

// All settled (success or failure)
await Promise.allSettled([p1, p2, p3]);

Advanced Patterns

// Retry with exponential backoff
async function retryWithBackoff(fn, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      return await fn();
    } catch (err) {
      if (i === retries - 1) throw err;
      await new Promise(r => setTimeout(r, 1000 * 2 ** i));
    }
  }
}

Common Pitfalls

  • Forgetting to handle promise rejections (unhandled rejection)
  • Creating "floating" promises without proper error handling
  • Using sequential awaits when parallel execution is possible
  • Not cleaning up promises in component lifecycle (memory leaks)
  • Ignoring promise states in conditional logic
